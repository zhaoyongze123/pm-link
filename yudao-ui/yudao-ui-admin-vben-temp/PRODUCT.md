# Product

## Register

product

## Users

这是一套给企业内部员工、审批人、流程管理员共同使用的审批工作台。普通用户需要在最短路径里发起审批、追踪进度、查看通知；管理员需要在同一套界面里进入流程、表单、分类、用户分组等管理能力，但不应被额外的前端壳子打断。

## Product Purpose

产品的目标不是展示“系统很多”，而是让审批这件事在一个统一工作台里变得清晰、可追踪、可处理。成功标准是三件事：用户能快速找到可发起的流程，审批人能稳定处理待办，管理员能通过权限动态进入系统管理与流程配置，而不用切换到另一套视觉体系。

## Brand Personality

克制、可靠、专业。语气应像成熟的企业协作产品，而不是营销页或设计练习。界面需要传达秩序感、响应感和可信度，让用户愿意在高频办公场景里长时间停留。

## Anti-references

- 不要做成“AI 默认后台”：蓝紫渐变、玻璃卡片、泛滥的圆角、大量阴影、每块都像独立组件。
- 不要做成营销页：夸张大标题、情绪化文案、装饰性动效、无意义的视觉噱头。
- 不要做成多套系统拼接：用户侧、管理侧、通知侧各用一套不同语言。

## Design Principles

- 统一壳子优先于局部修饰，同一导航、同一层级、同一组件语法覆盖用户与管理场景。
- 让审批任务优先于装饰，视觉重点始终落在待办、状态、流程信息和下一步动作。
- 熟悉感优先于奇观，组件应像成熟办公产品，而不是“重新发明”交互。
- 权限决定功能显隐，前端不写死管理员结构，视觉系统也不暗示双套身份。
- 明暗主题必须等价，不允许暗色模式只是浅色反相或局部补丁。

## Accessibility & Inclusion

至少满足常规办公场景的 WCAG AA 对比度要求。明暗主题都必须保证正文、标签、按钮、表单、状态色可读。动效只用于状态反馈，尊重 reduced motion。高频列表、表格、表单和图标按钮都要保持清晰点击目标与键盘焦点可见性。
